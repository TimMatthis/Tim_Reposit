Calibrated prediction for a new system

Turns a simulated profit from the optimiser into an expected actual profit with a reliability band, using the persisted fleet behaviour-segment model and the lowest-MAE calibration model.

What is a “primary segment”?

The reference fleet is split into K=4 behaviour segments by k-means on standardised usage features — average load, evening load share, import/export/battery activity, price spread, solar self-consumption and export:import ratio. Each cluster is given a stable label S1…S4, ordered by the cluster's average net margin in the reference fleet (S1 lowest, S4 highest).

A new home's features are z-scored with the reference fleet's mean/std and the distance to each centroid is computed. The primary segment is the nearest centroid — the fleet “shadow” the home most resembles. The secondary segment is the second nearest; if the two distances are near-equal the home is flagged ambiguous and soft weights are used to blend segment-specific calibrations. The primary segment's own sim→actual calibration and in-segment residuals drive both the point estimate and the p10–p90 reliability band, making the band personalised to homes that behave like this one rather than to the fleet as a whole.

Result

Primary segment = the fleet behaviour cluster this home most resembles (S1 = lowest-margin shadow, S4 = highest). The reliability band below is built from the sim→actual residuals of other homes in the same segment.

Diagnostics

Primary = nearest centroid in z-space. Secondary = second nearest (used to detect ambiguity). Soft weights blend segments when the home sits between clusters. Out-of-distribution fires when the primary distance exceeds the fleet's p90, causing the reliability band to be widened or fall back to fleet-wide residuals.
