Uploads & photo check

Bills and PDFs stay on whatever CMS or storage you use. Photo checking calls the small Node utility API (vision + prompt in the repo). If you open this page from a static server on port 8080, that server does not implement /api — point the field below at the utility server (usually port 3847) and run Test connection.

API for photo check

Analyze requests go to …/api/no-bill/analyze-photos. Resolve URL query overrides: ?api=http://127.0.0.1:3847 or ?utilityPort=3847.

Will call: —

  1. Bills

    Last four quarters or annual summary (PDF). Wire to HubSpot or your store.

    Bill / PDF upload placeholder

  2. Site photos

    JPG, PNG, or WebP — up to 8 files. Images are resized in the browser before upload.

Technical notes

Vision prompt: utility_api/noBillPhotoPrompt.js. Start API: cd utility_api && node server.js. Optional: serve everything from that server on 3847 so the base URL can be same-origin (empty path). OPENAI_API_KEY enables live model output; without it, the API returns structured mock JSON.
